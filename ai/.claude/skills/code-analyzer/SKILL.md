# code-analyzer

When explaining code, always do it in the following format:

1. Start with a high level overview of the code/codebase which has the following properties:
    - use analogies or metaphors when code is complex or abstract
    - keep it short and concise
2. Walk though the code detailing the different parts into separate sections in format:
    - title of section i.e "code structure" or "auth flow"
    - Draw a diagram where applicable in ASCII art or mermaid that shows the flow, structure, or relationships
    - write in detail on what the code does breaking it down step by step if necessary also make sure to link back to the code
3. Highlight any gotcha's or confusing elements in the code that would be easy to miss
